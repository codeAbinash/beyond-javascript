History of Javascript

JavaScript was created by Brendan Eich in 1995 just in 10 days. Became the ECMA-262 standard in 1997. It was originally named Mocha and later known as JavaScript. It was developed for Netscape. Now Mozilla foundation continued to develop JavaScript for the Firefox browser.

Javascript Engine

It is a software that executes JavaScript code. JavaScript engine is a part of modern browsers, when javascript was invented, it was a interpreted language. But now modern browsers use just in time (JIT) compilation to improve performance. JavaScript engine is used not only in browsers but also it is a core component of Node.js. Example of some famous JavaScript engines are -

  • V8 engine by Google for Google Chrome
  • Spider monkey by Mozilla for Firefox Browser
  • JavaScriptCore by Apple for Safari Browser
  • Chakra by Microsoft for Internet Explorer
  • Chakra was used by Microsoft Edge browser, but now it uses V8 engine

What happens in JavaScript engine?

JavaScript engine converts high level code into machine readable code. Different JavaScript engines use different techniques to execute JavaScript code. Some engines use interpreter and compiler both (e.g. V8 engine). As an example, Google chrome's JavaScript V8 engine works like this -

First JavaScript source code goes to Parser. Manly it checks for syntax. Breaks the codes into token.

Now these tokens are converted to AST(Abstract Syntax Tree) it allows interpreter to read the code and understand.

Now AST goes to Interpreter to get converted into Byte code. In V8 engine it is known as Ignition.

If there is repeating code, it will decrease the performance of interpreter. Profiler will check repeating codes. To increase performance Profiler moves repeating code to compiler.

Compiler generates byte code. Byte code executes to do specific task. In v8 engine this compiler is called as TurboFan.

To know more about it just visit Google's v8 engine Blog.

Syntax

Syntax of a computer language is the set of rules that defines the combinations of symbols that are considered to be correctly structured statements or expressions in that language. JavaScript Syntax is similar to C, C++.

Expressions

JavaScript Expressions Any unit of code that can be evaluated to a value is an expression.

Statement

A statement is a syntactic unit of an imperative programming language that expresses some action to be carried out. Simply, a statement is complete part of code commanding a task. JavaScript statements ends with Semicolon. But it is optional (using semicolon at the end of each statement is recommended by many developers).

If we have to write more than one statement in one line, we have to use semicolon to separate each statement. Otherwise it is optional.

JavaScript ignores multiple whitespace between statements. Using one whitespace makes the code more readable.

To break a long statement, best practice to break it after an operator.

Keywords

JavaScript Keywords are predefined, reserved words used in a program that have special meanings to the compiler or interpreter. Some examples of javascript keywords var , let , if , else , while. See all Keywords.

Identifiers

An identifier is a sequence of characters in the code that identifies a variable, function, or property. In JavaScript, identifiers are case-sensitive and can contain Unicode letters, $, _, and digits 0-9 but can not start with a digit.

Block or Code Block

To group multiple JavaScript statements , we have to use { } curly brackets. Code inside those curly brackets are called Block or Code Block. Code inside curly brackets { code } works like a single statements. We can use block to group multiple statements or zero statement into one where JavaScript expects single statement. If we need not to provide any statement (zero statement), we can use empty block.
