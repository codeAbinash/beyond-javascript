Temporal Dead Zone

When a variable is declared using let, before executing a single line of code, the variable is stored in memory and undefined is stored in it as initial value. But it cannot be accessed before declaration. From the starting of the scope (when the variable is stored in memory) until the declaration is done, is called "Temporal Dead Zone (TDZ)" of that variable. Variables cannot be accessed in TDZ. Accessing them will cause an error (ReferenceError).

const

Const means constant. Variables declared using const are similar to the variables declared using let. The main difference is, constants cannot be reassigned using assignment (=) operator. They must be assigned in the same statement in which they are declared.

My favorite color is Purple

const variable can be declared using the same characters used for let and var. But traditional way to do it using all UPPERCASE_LETTERS for better code readability.

Must be Assigned while declaring

Variables declared using const keyword, must be assigned in the same statement in which it is declared otherwise it will throw an error (SyntaxError).

SyntaxError: Missing initializer in const declaration

Reassigning : error

Reassigning variables using const keyword will trow an error (TypeError).

TypeError: Assignment to constant variable

const : Constant reference to a value

const does not mean a constant value. It defines a constant reference to a value. That means, that reference cannot be changed.
