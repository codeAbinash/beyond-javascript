Variable declaration using var

Variables declared using var keyword are function-scoped (accessible only in a function in which it is declared) or global-scoped (accessible everywhere in the program).

Hoisting

When a variable is declared using var keyword, before executing a single line of code, this variable is stored in Global scope or function scope and a special value undefined is stored in it as it's initial value. This is called Hoisting in JavaScript.

Access Before Declaring

Variables, declared with var keyword are hoisted. So they can be accessed before declaring. Because those variables are already stored in global scope or function scope before execution.

Without Declaring

If you directly initialize a variable without declaring it (e.g. a = 10 ), JavaScript will not throw any error. These variables are treated as global variable. Unlike variables declared using var keyword, these variables are NOT hoisted. Memory allocation of these variables happen while executing. So if we want to access a variable before initialization, JavaScript will throw an error (ReferenceError). It is not recommended to use this type of initialization. You should always declare a variable formally using const, let or var keyword.
