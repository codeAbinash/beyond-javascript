Comparison Operator

Comparison Operator compares it's operand and returns true or false (logical value). If two operands are not same type, JavaScript tries to convert them to similar type for comparison. There are strict operators in JavaScript (=== & !==) which do not convert the operands for comparison.

  • Equality
  • Inequality
  • Strict Equality
  • Strict Inequality
  • Greater Than
  • Greater Than or Equal
  • Less Than or Equal

Inequality(!=)

The inequality operator checks if it's two operands are not equal. It returns true if it's operands are not equal else it returns false. It tries to convert it's operands if they are not of same type.

Inequality operator does the negation of the equality operator. You can say it is the opposite operator of Equality Operator

a != b is equivalent to !(a == b)

Check logical operators for more details about !(logical not) operator.


Strict Equality ( === )

Strict Equality operator checks if it's two operands are qual. If the type of the operands are not same, it returns false. It will not convert the type of the operands.

If the operands are of same type, it will return true, if the value of the both operand are same. Else it will return false.

Strict Inequality ( !== )

Strict Inequality operator does the negation of Strict Equality operator. You can say this is the opposite of Strict Equality Operator.

a !== b is equivalent to !(a === b)

Abstract Relation Comparison

The Abstract Relation Comparison algorithm defines how the comparison works in JavaScript.

• If Both operands are not string

It tries to convert both operand into number and then compares.

  • If one or both operands are converted to NaN return false.
  • If Numerical value of the first and the second operand are same, return false.
  • +Infinity and -Infinity case.
  • If the mathematical value of the first operand is greater than the mathematical value of second operand, returns true, else returns false. (Both operands are finite)

• If Both operands are string

  • If first operand is prefix of second operand, return false.

    A string value p is a prefix of string q iff q == p + r is true, r is another string.

    A string is a prefix of itself because s == s + "" is true.

  • If second operand is prefix of first operand, return true.
  • Random case. String values are compared using a simple lexicographic ordering on the sequence of UTF-16. Let's explore some examples for better understanding.

Greater Than or Equal (>=)

Greater than or Equal operator returns true if it's left operand is greater than or equal to the right operand, else returns false. It works based on the similar type Abstract Relation Comparing algorithm.

Less Than or Equal (<=)

Less than or Equal operator returns true if it's left operand is less than or equal to the right operand, else returns false. It works based on the similar type Abstract Relation Comparing algorithm.
