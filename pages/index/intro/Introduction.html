Object Oriented

Object Oriented Programming is a programming model based on objects having four basic concepts of programming - Encapsulation, Abstraction, Inheritance and Polymorphism.. Read more about Object Oriented Programming on Wikipedia.

Single Threaded

JavaScript has only one call stack and one memory heap. So it can execute only one thing at a time. That makes JavaScript a single threaded programming language. Read more on Wikipedia.

Dynamic

JavaScript is a Dynamically Typed Language that means data type of a variable can be changed in runtime. Read more on Wikipedia.

High Level

If a programming language is more user friendly, independent of computer's hardware architecture, it is called a High Level programming language. Read more on Wikipedia.

Just in time compiled

Just in time compilation means the compilation is done during the execution of the code. Read more on Wikipedia

Prototype-Based

Prototype-based programming is a style of object-oriented programming in which classes are not explicitly defined, but rather derived by adding properties and methods to an instance of another class or, less frequently, adding them to an empty object. (From MDN).

Multi-Paradigm

Multi-Paradigm Supporting more than one programming paradigm, in order to allow the most suitable programming style for a task. Read more on Wikipedia.

Functional Programming Style

It is a programming paradigm where programs are constructed by applying and composing function. Read more on Wikipedia.

Scripting Language

A scripting language or script language is a programming language for a runtime system that automates the execution of tasks that would otherwise be performed individually by a human operator.[1] Scripting languages are usually interpreted at runtime rather than compiled. (on Wikipedia).

Imperative

In computer science, imperative programming is a programming paradigm of software that uses statements that change a program's state. ( on Wikipedia).

Lightweight

Javascript is a lightweight programming language. That means it uses less resources (memory , CPU etc.) to do it's job.

History of Javascript

JavaScript was created by Brendan Eich in 1995 just in 10 days for Netscape Browser. Became the ECMA-262 standard in 1997. It was originally named Mocha and later known as JavaScript. Now Mozilla foundation continued to develop JavaScript for the Firefox Browser.

Javascript Engine

It is a software that executes JavaScript code. JavaScript engine is a part of modern browsers, when javascript was invented, it was an interpreted language. But now modern browsers use just in time (JIT) compilation to improve performance. JavaScript engine is used not only in browsers but also it is a core component of Node.js. Example of some famous JavaScript engines are -

  • V8 engine by Google for Google Chrome
  • Spider monkey by Mozilla for Firefox Browser
  • JavaScriptCore by Apple for Safari Browser
  • Chakra by Microsoft for Internet Explorer
  • Chakra was used by Microsoft Edge browser, but now it uses V8 engine

What happens inside JavaScript engine?

JavaScript engine converts high level code into machine readable code. Different JavaScript engines use different techniques to get the job done. Some engines use interpreter and compiler both (e.g. V8 engine).

Google chrome's JavaScript V8 engine works like this -

First JavaScript source code goes to Parser. Mainly it checks for syntax. Breaks the codes into token.

Now these tokens are converted to AST(Abstract Syntax Tree) it allows interpreter to read the code and understand.

Now AST goes to Interpreter to get converted into Byte code. In V8 engine it is known as Ignition.

If there is repeating code, it will decrease the performance of interpreter. Profiler will check for repeating codes. To increase performance of execution Profiler moves repeating code to compiler.

Compiler generates byte code. Byte code executes to do specific task. In V8 engine this compiler is called as TurboFan.

To know more about it just visit Google's V8 engine Blog.

Syntax

Syntax of a computer language is the set of rules that defines the combinations of symbols that are considered to be correctly structured statements or expressions in that language. JavaScript Syntax is similar to C, C++.

Comments

Comments are not executed by JavaScript. They are ignored. Comments in JavaScript are same as in C++.

Nested comments are not allowed. /* Comment1 /* Comment2 */ Comment3 */

Expressions

Any unit of code that can be evaluated to a value is an expression.

Statement

A statement is a syntactic unit of a programming language that expresses some action to be carried out. Simply, a statement is complete part of code commanding a task. JavaScript statements end with Semicolon. But it is optional (using semicolon at the end of each statement is recommended by many developers).

If we have to write more than one statement in one line, we have to use semicolon to separate each statement. Otherwise it is optional.

JavaScript ignores multiple whitespace. Using one whitespace makes the code more readable.

To break a long statement, best practice to break it after an operator.

Keywords

JavaScript Keywords are predefined, reserved words used in a program that has special meanings to the compiler or interpreter. Some examples of javascript keywords are : var , let , if , else , while etc. See all Keywords.

Identifiers

An Identifier is a sequence of characters in the code that identifies a variable, function, or property. In JavaScript, identifiers are case-sensitive and can contain Unicode letters, $, _, and digits 0-9 but can not start with a digit.

Block or Code Block

To group multiple JavaScript statements , we have to use { } curly brackets. Code inside those curly brackets are called Block or Code Block. Code inside curly brackets { code } works like a single statements. We can use block to group multiple statements or zero statement into one where JavaScript expects single statement. If we need not to provide any statement (zero statement), we can use empty block.
