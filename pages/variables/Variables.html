JavaScript Variables

What is variable

Variables are names given to a memory location to store data in a program and use it in the program where it is needed. Simply a variable is a container for a value (data).

Variable Naming Rule

Variable names -

  • can contain letters (A-Z, a-z), numbers (0-9), dollar sign ($) and underscore (_).
  • Must start with a letter, underscore or dollar sign.
  • Must not start with a number.
  • Cannot use reserved words e.g. let while etc.
  • Must be a single word, cannot use space (or any whitespace)
  • JavaScript allows naming variables using most of unicode characters, but it is not recommended

Declaring a Variable

To use a variable in a program we have to declare it first, to let the JS Engine know we have to allocate a memory space to store a data in it.

There are 3 types of variable declaration in JavaScript. Using var, let and const.




Variable declaration using var

Variables declared using var keyword are function-scoped (accessible only in a function in which it is declared) or global-scoped (accessible everywhere in the program).

When a variable is declared using var keyword, before executing a single line of code, this variable is stored in Global scope or function scope and a special value undefined is stored in it as it's initial value. This is called Hoisting in JavaScript.

Without Declaring

If we directly initialize a variable without declaring it (e.g. a = 10 ), JavaScript will not throw any error. These variables are treated as global variable. Unlike variables declared using var keywords, these variables are NOT hoisted. Memory allocation of these variables happen while executing. So if we want to access a variable before initialization, JavaScript will throw an error (ReferenceError) . Because there is no variable present in memory(or required scope) as that desired variable name. It is not recommended to use this type of initialization. You must declare a variable before initialization.

Temporal Dead Zone

When a variable is declared using let, before executing a single line of code, the variable is stored in memory and undefined is stored in it as initial value. But it cannot be accessed before declaration. From the starting of the scope (when the variable is stored in memory) until the declaration is done, is called "Temporal Dead Zone (TDZ)" of that variable. Variables cannot be accessed in TDZ. Accessing them will cause an error (ReferenceError).

const

Const means constant. Variables declared using const are similar to the variables declared using let. The main difference is, constants cannot be reassigned using assignment (=) operator. They must be assigned in the same statement in which they are declared.

My favorite color is Purple

const variable can be declared using the same characters used for let and var. But traditional way to do it using all UPPERCASE_LETTERS for better code readability.

Must be Assigned while declaring

Variables declared using const keyword, must be assigned in the same statement in which it is declared otherwise it will throw an error (SyntaxError).

SyntaxError: Missing initializer in const declaration

Reassigning : error

Reassigning variables using const keyword will trow an error (TypeError).

TypeError: Assignment to constant variable

const : Constant reference to a value

const does not mean a constant value. It defines a constant reference to a value. That means, that reference cannot be changed.
