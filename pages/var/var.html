Variable declaration using var

Variables declared using var keyword are function-scoped (accessible only in a function in which it is declared) or global-scoped (accessible everywhere in the program).

Hoisting

When we declare a variable using var, before executing a single line of code, those variables are stored in Global scope or function scope and a special value undefined is stored in it as their initial value. This is called Hoisting in JavaScript.

Use Before Declaring

Variables, declared with var keyword are hoisted. So they can be used before initialization. Because those variables are already stored in global scope or function scope.

Without Declaring

If we declare variables using nothing, for example a = 100 JavaScript will not throw any error. These variables will be treated as global variable. Unlike var declared variables, Memory allocation of these variables happens while executing. So if we want to access this variables before initializing, JavaScript will throw an error (ReferenceError) . Because JavaScript cannot find anything as that name of variable in memory (any scope). It is not recommended to use this type of declaration.
